# Загрузка справочника из Excel — инструкция пользователя

Внешняя обработка для BAS Бухгалтерія КОРП 2.1 (управляемые формы). Позволяет
загрузить или обновить элементы любого справочника конфигурации из файла
Excel с гибкой настройкой соответствия колонок реквизитам.

Файл обработки: `ЗагрузкаСправочникаИзExcel.epf`

---

## 1. Подключение обработки

### Вариант А — через «Дополнительные отчёты и обработки» (рекомендуется)

1. Откройте раздел **НСИ и администрирование → Печатные формы, отчёты и
   обработки → Дополнительные отчёты и обработки**.
2. Нажмите **Создать**, укажите файл `ЗагрузкаСправочникаИзExcel.epf`.
3. Настройте права доступа для нужных пользователей/групп.
4. После подключения обработка появится в общем списке дополнительных
   обработок под названием «Загрузка справочника из Excel».

Если ранее обработка уже была подключена и вы получили новую версию файла —
**удалите старую запись и подключите файл заново** (или используйте команду
обновления из файла, если она доступна в вашей версии БСП). Платформа хранит
собственный снимок файла внутри базы и не подхватывает изменения автоматически.

### Вариант Б — напрямую

**Файл → Открыть** → выбрать `ЗагрузкаСправочникаИзExcel.epf`. Подходит для
разового запуска администратором, без регистрации в списке дополнительных
обработок.

---

## 2. Подготовка файла Excel

- Первая строка файла — **заголовки колонок**.
- Каждая последующая строка — один элемент справочника.
- В поле «Колонка файла» (см. п. 3–4) можно указать **либо букву колонки**
  (`A`, `B`, `C`…), **либо точный текст заголовка** из первой строки —
  работает и то, и другое. При указании заголовка текст должен совпадать
  с ячейкой файла дословно (с учётом регистра и пробелов).
- Поддерживаемые форматы: `.xlsx`, `.xls`, `.ods`.
- **Ссылочные реквизиты и «Родитель»** заполняются **кодом** существующего
  элемента/группы связанного справочника (а не наименованием и не ссылкой).
  Если элемент/группа с таким кодом не найдена — строка считается ошибочной.
- **Реквизиты табличных частей**: значения для одной строки табличной части
  через **точку с запятой `;`** в одной ячейке. Количество элементов в
  списках разных колонок одной табличной части должно соответствовать
  количеству строк, которые нужно получить (по самому длинному списку).
  Пример для табличной части «Контакты» с колонками «Вид» и «Значение»:
  - ячейка «Вид»: `Телефон;Email`
  - ячейка «Значение»: `+380501234567;test@example.com`

  Получится 2 строки табличной части.
- Даты указывайте в формате `ГГГГ-ММ-ДД` (например `2026-06-27`).
- Логические значения: `Да`/`Истина`/`1` — истина, любое другое значение —
  ложь.
- Если для реквизита **не нужна** колонка файла (одно и то же значение для
  всех строк или просто нет такого поля в файле), можно задать **значение
  по умолчанию** прямо в таблице сопоставления — см. п. 3.

## 3. Интерфейс формы

| Поле / элемент | Назначение |
|---|---|
| **Вид справочника** | Выбор справочника конфигурации, элементы которого нужно загрузить |
| **Файл данных** | Путь к файлу Excel (кнопка «…» открывает диалог выбора) |
| **Способ чтения** | «Без Excel (платформенно)» — работает без установленного MS Excel; «Через MS Excel (COM)» — требует установленный MS Excel на компьютере пользователя, толстый клиент |
| **Таблица сопоставления** | Список всех доступных реквизитов выбранного справочника (включая ссылочные, «Родитель», реквизиты табличных частей) |
| → колонка «Колонка файла» | Укажите, какой колонке файла соответствует реквизит — буквой (`A`, `B`…) или точным текстом заголовка из первой строки файла |
| → колонка «По умолчанию» | Значение (или код — для ссылочных реквизитов), которое подставляется, **если колонка файла не указана**. Если не указаны ни колонка, ни значение по умолчанию — реквизит остаётся незаполненным |
| → колонка «Ключ поиска» | Отметьте флажком реквизит(ы), по которым искать существующий элемент перед записью. Для ключевых реквизитов значение «По умолчанию» не используется — нужна именно колонка файла |
| **Таблица анализа** | Результат проверки файла по строкам: что найдено в базе, что можно создать/обновить, и что в итоге сделано |
| → колонка «Обработать» | Флажок — включает строку в обработку при нажатии «Создать»/«Обновить». Можно снять флажок у отдельных строк, чтобы пропустить их |
| → колонка «Статус» | Текущее состояние строки (см. таблицу статусов ниже) |
| → колонка «Элемент» | Найденная ссылка на элемент справочника, либо «(новый)», если элемент не найден |
| → колонка «Детали» | Для «Можно обновить» — какие реквизиты отличаются; для «Ошибка» — причина |

Под таблицей анализа — три кнопки: **«Проанализировать»**, **«Создать»**,
**«Обновить»**.

## 4. Порядок работы

Процесс двухшаговый: сначала **анализ без записи в базу**, затем — раздельное
создание новых элементов и обновление существующих.

### Шаг 1 — настройка и анализ

1. Выберите **вид справочника**.
2. В таблице сопоставления для каждого нужного реквизита укажите **колонку
   файла** либо **значение по умолчанию** (если колонка не нужна — например,
   одно и то же значение для всех строк). Реквизиты без колонки и без
   значения по умолчанию в загрузке участвовать не будут.
3. Отметьте **хотя бы один** реквизит как **«Ключ поиска»** — по нему
   обработка будет проверять, есть ли элемент уже в базе. Можно отметить
   несколько реквизитов сразу (поиск тогда идёт по их сочетанию).
4. Укажите **файл данных** и **способ чтения**.
5. Нажмите **«Проанализировать»**. Обработка прочитает файл и для каждой
   строки (без какой-либо записи в базу) определит статус:
   - **Можно создать** — элемент по ключу не найден, будет создан новый;
   - **Можно обновить** — элемент найден, но есть отличия от данных файла
     (в «Деталях» — какие именно реквизиты отличаются);
   - **Без изменений** — элемент найден, все сопоставленные значения уже
     совпадают, действие не требуется;
   - **Ошибка** — строка не может быть обработана (в «Деталях» — причина: не
     заполнен ключ, не найден код связанного справочника и т. п.).

   Строки со статусом «Можно создать» и «Можно обновить» автоматически
   отмечены флажком **«Обработать»** — снимите флажок у тех строк, которые
   нужно пропустить.

### Шаг 2 — создание и обновление

6. Нажмите **«Создать»** — будут созданы элементы для всех отмеченных строк
   со статусом «Можно создать». После создания статус строки меняется на
   **«Создан»**, в «Элемент» появляется ссылка на новый элемент.
7. Нажмите **«Обновить»** — обработка заново пересчитает расхождения по всем
   отмеченным строкам со статусом «Можно обновить» и покажет **один общий
   диалог «Подтверждение расхождений»** со списком отличий **сразу по всем
   элементам** (колонки: «Строка», «Элемент», «Реквизит», «В базе», «В
   файле», флажок «Применить» у каждой строки списка). Снимите флажки у тех
   отличий, которые применять не нужно, и нажмите «Применить отмеченное»
   (или «Не применять ничего», чтобы отменить обновление для всех элементов
   сразу). После обработки статус каждой строки в таблице анализа меняется
   на:
   - **Обновлён** — применено хотя бы одно подтверждённое изменение (в
     «Деталях» — какие реквизиты изменены);
   - **Пропущен (расхождения отклонены)** — по этому элементу не было
     отмечено ни одного изменения в диалоге;
   - **Ошибка** — не удалось записать элемент (см. «Детали»).

Кнопки «Создать» и «Обновить» можно нажимать в любом порядке и по нескольку
раз — они обрабатывают только строки, у которых сейчас выставлен флажок
«Обработать» (уже обработанные строки автоматически снимают флажок сами).

**Важно:** «Создать» и «Обновить» при каждом нажатии заново читают файл по
тому же пути (а не запоминают данные из шага анализа). Поэтому между
«Проанализировать» и «Создать»/«Обновить» **не перемещайте, не переименовывайте
и не редактируйте** файл — состав и порядок строк должны остаться такими же,
как при анализе.

Файл можно анализировать и загружать **повторно** — это безопасно:
существующие элементы не дублируются, повторный анализ просто покажет «Без
изменений» там, где данные не поменялись.

## 5. Ограничения

- Группы и связанные элементы по ссылочным реквизитам и «Родитель»
  **не создаются автоматически** — они должны существовать в базе заранее.
  Если код не найден, строка помечается как ошибка.
- Реквизиты табличных частей сравниваются с базой **только при создании**
  нового элемента. При обновлении существующего элемента табличная часть не
  трогается (сравнение и обновление строк ТЧ не реализовано в этой версии).
- Разделитель для значений табличной части — фиксированный символ `;`.
- Способ чтения «Через MS Excel» работает только в толстом клиенте с
  установленным MS Excel; если COM недоступен — обработка покажет
  предупреждение, без чтения файла.
- Загружаются только справочники (документы, регистры — не поддерживаются).

## 6. Частые проблемы

| Сообщение | Что делать |
|---|---|
| «Выберите вид справочника» / «Выберите файл данных» | Заполните соответствующее поле перед запуском |
| «Отметьте хотя бы один реквизит как «Ключ поиска»» | Поставьте флажок в колонке «Ключ поиска» хотя бы у одной строки таблицы сопоставления |
| «Сначала выполните анализ (кнопка «Проанализировать»)» | Нажмите «Создать»/«Обновить» только после «Проанализировать» — без анализа обработка не знает, какие строки обрабатывать |
| «Нет отмеченных строк со статусом «Можно обновить»» | Либо нет строк с расхождениями, либо у них снят флажок «Обработать» |
| Строка в таблице анализа — «Ошибка», деталь «Не найдено значение ключа…» | В файле для этой строки пустая ячейка в колонке, сопоставленной с ключевым реквизитом |
| Строка в таблице анализа — «Ошибка», деталь про код справочника | Код связанного справочника/группы из файла не найден в базе — проверьте код или создайте элемент/группу заранее |
| Строка в таблице анализа — «Ошибка», деталь содержит «Возможно, элемент с таким же наименованием уже есть в базе: код "…"» | Обработка нашла в базе элемент с точно таким же «Наименование», но под другим кодом. Скорее всего, это тот же товар — укажите подсказанный код в файле (в колонке, сопоставленной с «Код») и запустите «Обновить» вместо «Создать» |
| «Чтение через MS Excel недоступно…» | Используйте способ «Без Excel» либо запустите обработку в толстом клиенте с установленным MS Excel |
