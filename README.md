# Загрузка справочника из Excel — внешняя обработка для BAS Бухгалтерія КОРП

Внешняя обработка (`.epf`) для **BAS Бухгалтерія КОРП 2.1** (платформа BAF
8.3.23, управляемые формы). Позволяет загрузить или обновить элементы
**любого справочника** конфигурации из файла Excel с гибкой настройкой
соответствия колонок реквизитам — без программирования под конкретный
справочник.

Готовый файл: [`erf/ЗагрузкаСправочникаИзExcel.epf`](erf/ЗагрузкаСправочникаИзExcel.epf)

Подробная инструкция пользователя: [`erf/Инструкция_пользователя.md`](erf/Инструкция_пользователя.md)
/ [`erf/Инструкция_пользователя.docx`](erf/Инструкция_пользователя.docx)

---

## Возможности

- Выбор любого справочника конфигурации, динамическое построение списка
  доступных реквизитов (простые, ссылочные, `Родитель` для иерархических
  справочников, реквизиты табличных частей, **Перечисление**).
- Сопоставление каждого реквизита с колонкой файла — буквой (`A`, `B`…) или
  точным текстом заголовка, либо **значением по умолчанию**, если колонка не
  нужна.
- Два способа чтения файла: платформенно (`ТабличныйДокумент.Прочитать()`,
  без MS Excel) или через COM `Excel.Application` (толстый клиент).
- Двухшаговый процесс: **«Проанализировать»** — предпросмотр без записи в
  базу (что можно создать / обновить / уже совпадает / ошибка), затем
  раздельные **«Создать»** и **«Обновить»** с явным выбором, какие строки
  обрабатывать.
- Перед обновлением найденного элемента — единый диалог подтверждения со
  списком всех расхождений по всем элементам сразу, с отметкой применения
  по каждому реквизиту.
- Ссылочные реквизиты и «Родитель» сопоставляются по **коду** связанного
  справочника/группы (без автосоздания отсутствующих).
- Реквизиты табличных частей заполняются списком значений через `;` в одной
  ячейке (только при создании нового элемента).
- При ошибке уникальности — подсказка, если в базе уже есть элемент с таким
  же наименованием под другим кодом.
- Подключается через подсистему БСП «Дополнительные отчёты и обработки»,
  без модификации конфигурации.

## Структура репозитория

```
src_epf/    — XML-исходники внешней обработки (для сборки через 1С Designer)
erf/        — собранный .epf + инструкция пользователя (md и docx)
openspec/   — спецификация и план разработки (OpenSpec: proposal, design, specs, tasks)
```

## Подключение и использование

См. [`erf/Инструкция_пользователя.md`](erf/Инструкция_пользователя.md).

Кратко: НСИ и администрирование → Дополнительные отчёты и обработки →
Создать → выбрать `erf/ЗагрузкаСправочникаИзExcel.epf`.

## Ограничения

- Группы и связанные элементы по ссылочным реквизитам и «Родитель» должны
  существовать в базе заранее — автосоздание не выполняется.
- Реквизиты табличных частей сравниваются с базой только при создании нового
  элемента (при обновлении существующего табличная часть не трогается).
- Разделитель значений табличной части — фиксированный символ `;`.
- Чтение через MS Excel (COM) работает только в толстом клиенте с
  установленным MS Excel.

## Среда разработки и тестирования

- Платформа: BAF 8.3.23.2299 (используемая в проекте альтернативная сборка
  платформы 1С:Предприятие).
- Конфигурация: BAS Бухгалтерія КОРП, редакція 2.1 (2.1.5.4).
- Дополнительно проверено в продуктивной базе BAS КУП 2.1.22.2 (платформа
  8.3.18, MS SQL).
