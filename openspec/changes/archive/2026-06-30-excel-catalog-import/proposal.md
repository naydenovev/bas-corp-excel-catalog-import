## Why

В BAS Бухгалтерія КОРП 2.1 нет универсального инструмента первоначального
заполнения справочников из Excel-выгрузок (от поставщиков, из старых баз,
из прайс-листов). Ручной ввод элементов справочников (Номенклатура,
Контрагенты, Номенклатура.Характеристики и т.п.) с большим количеством строк
неэффективен и не отказоустойчив к повторному запуску (дублирование
элементов). Нужна внешняя обработка, которая позволяет загрузить произвольный
справочник из Excel-файла с гибким сопоставлением колонок реквизитам,
безопасной проверкой дублей по выбираемому пользователем ключу и контролируемым
обновлением уже существующих элементов.

## What Changes

- Новая внешняя обработка `ЗагрузкаСправочникаИзExcel.epf`, подключаемая через
  подсистему БСП «Дополнительные отчёты и обработки» (тип — «Заполнение
  данных» или «Вызов из списка/произвольная команда», без модификации
  конфигурации).
- Управляемая форма с выбором вида справочника (любой справочник конфигурации
  через `Метаданные.Справочники`).
- Динамическое построение таблицы соответствия «Реквизит ↔ Колонка Excel» по
  факту выбора справочника, включая простые реквизиты, ссылочные реквизиты,
  реквизит `Родитель` (для иерархических справочников) и реквизиты табличных
  частей.
- Два режима чтения исходного файла с выбором на форме:
  - через `ТабличныйДокумент.Прочитать()` — без COM, доступно в тонком и
    толстом клиенте;
  - через COM `Excel.Application` — только в толстом клиенте с установленным
    MS Excel.
- Отметка пользователем одного или нескольких сопоставленных реквизитов как
  «ключ поиска» — по нему выполняется проверка наличия элемента в базе перед
  записью.
- Сопоставление ссылочных реквизитов и `Родитель` — по коду существующего
  элемента/группы связанного справочника (без автоматического создания
  отсутствующих групп/элементов; несовпадение кода — ошибка строки).
- Заполнение табличных частей справочника: одна строка Excel — один элемент
  справочника, значения для каждой строки табличной части передаются списком
  через разделитель в одной ячейке.
- Перед записью найденного по ключу элемента — сравнение значений из Excel со
  значениями в базе; при наличии расхождений — диалог с перечнем расхождений и
  явным подтверждением, какие изменения применять; решения и фактически
  применённые изменения фиксируются в протоколе загрузки (отдельный текстовый
  отчёт/табличный документ на форме).
- Если элемент по ключу не найден — создание нового элемента с заполнением
  всех сопоставленных реквизитов.

## Capabilities

### New Capabilities
- `excel-catalog-import`: внешняя обработка для загрузки/обновления элементов
  произвольного справочника BAS Бухгалтерія КОРП из Excel-файла с настраиваемым
  сопоставлением колонок, выбором ключа поиска дублей, поддержкой иерархии,
  табличных частей и контролируемым обновлением существующих элементов с
  ведением протокола.

### Modified Capabilities
(нет — типовая конфигурация не изменяется, обработка подключается как внешняя
через стандартный механизм БСП)

## Impact

- Новые файлы только во внешней обработке (`src_epf/` или аналогичная папка
  исходников EPF в `E:\project\BAS_corp`), конфигурация BAS не модифицируется.
- Точка интеграции с конфигурацией — только чтение метаданных
  (`Метаданные.Справочники`, реквизиты, табличные части) и стандартные вызовы
  записи объектов (`СправочникСсылка.ПолучитьОбъект()`,
  `Справочники.<Имя>.СоздатьЭлемент()`, `Записать()`), без правки модулей
  объектов конфигурации.
- Подключение через БСП «Дополнительные отчёты и обработки» — обработка
  регистрируется как дополнительная обработка, доступная пользователям с
  соответствующими правами на эту подсистему; права на сами справочники
  (создание/изменение элементов) типовые, не расширяются.
- Зависимость от наличия подсистемы БСП «Дополнительные отчёты и обработки» в
  конфигурации (присутствует в BAS Бухгалтерія КОРП 2.1).
- Опциональная зависимость от установленного MS Excel на клиентском месте —
  только при выборе режима чтения через COM.
